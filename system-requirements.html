Sequence

🔑 How to Read a Sequence Diagram

─────► SOLID ARROW
Synchronous call / request
Caller waits for response
- - - -► DASHED ARROW
Return value / response
Data flows back
│ LIFELINE │
Vertical line = actor exists
Thick box = active/processing

Diagram · User Flow

User Flow
Diagram

Shows the DECISIONS and PATHS a user takes. Rectangles = actions, Diamonds = decisions, Rounded = start/end. Click steps to walk through the flow.

🔑 User Flow vs Use Case vs Sequence

USE CASE
WHAT the system does. Goals & actors. High level narrative.
USER FLOW
WHERE the user goes. UI screens & decisions. Designer-friendly.
SEQUENCE DIAGRAM
HOW components talk. Messages & timing. Developer-friendly.

Diagram · Swimlane / Activity

Swimlane
Diagram

Shows WHO does WHAT. Each lane belongs to one actor. Actions flow across lanes. Perfect for showing handoffs between teams/systems.

🏊 Why Swimlane Diagrams?

When a process spans multiple actors (Customer, Frontend, Backend, Payment Service, Inventory), swimlanes make responsibility crystal clear. Every box has an owner. Every handoff is visible.
